# 基础配置（新手友好：只改这几处就够）

目标：让你在 **不破坏整体体验** 的前提下，改到最常用的 3～5 个开关。

> 你所有改动，优先写在用户目录根目录的 `wanxiang.custom.yaml`（或 `wanxiang_pro.custom.yaml`）。

## 0) 一条铁律：custom 文件顶层只保留一个 `patch:`

万象对 patch 机制有专门说明：`10-patch方法论.md`。

常见“改了不生效”的原因之一：把内容插进 `patch:` 中间导致段落被截断。

## 1) 候选数量（建议保持 6）

文件：`wanxiang.custom.yaml`

你会看到类似：

- `menu/page_size: 6`

万象默认建议不超过 6（因为 `7 8 9 0` 常用于声调/功能映射）。新手先别改大。

## 2) 开关记忆（切换窗口不重置）

文件：`default.yaml`

`switcher/save_options:` 列表决定哪些开关会被记住。你可以按需增删，但建议先只增加你确定常用的开关。

## 3) Tips（提示区）开关

文件：`wanxiang.schema.yaml`（定义开关） + `wanxiang.custom.yaml`（决定默认状态）

在 schema 里有 `switches`，其中 `super_tips` 默认 `reset: 1`（通常表示默认开启）。

如果你想默认关闭，可以在 `wanxiang.custom.yaml` 里 patch 对应项（建议先理解 `switches` 的结构再动）。

## 4) 简繁/翻译/Emoji（建议先学会“开关在哪里”）

你会在 `wanxiang.schema.yaml` 看到类似：

- `simplifier@emoji`
- `simplifier@s2t` / `s2hk` / `s2tw`
- `simplifier@chinese_english`

新手建议先通过“方案选单”切换它们，不要一上来就改过滤链顺序。

## 5) 自定义置顶/专有名词：用自定义词库（推荐）

轻量做法：编辑用户目录里的 `custom_phrase.txt`，写入你常用的词/短语。

更强做法：建立一个固定词库 `userxx.dict.yaml`，然后在 `wanxiang.custom.yaml` 里 `translator/packs/+` 引入（源样例里有注释提示）。
