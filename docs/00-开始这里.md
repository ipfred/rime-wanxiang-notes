# 开始这里

这份文档按“能用 → 好用 → 玩明白”的顺序组织；你可以从头读，也可以当目录索引用。

## 你现在属于哪种情况？

### A. 我是新手：想先打字成功

按这个顺序：

1. `01-5分钟上手.md`
2. `02-目录结构与核心文件.md`（知道文件位置就够了）
3. `03-基础配置.md`（只改 1～3 处常用项）

### B. 我已经能用了：想更顺手、可控

按这个顺序：

1. `03-基础配置.md`
2. `04-进阶配置.md`

### C. 我遇到问题：部署失败 / 不生效 / 词库不对

直接看：

- `05-常见问题.md`

## 文档约定（很重要）

- **“用户目录”**：Rime 真正读取配置的目录。不要在“你下载的方案包/源仓库”里的 `custom/` 目录直接改文件（那只是“携带样例/初始化来源”）。
- **“custom patch”**：你自己的最终配置入口，优先级最高，不会被更新覆盖。
- 本文档尽量引用源项目现有规则：当理解不清时，会提示你去看对应源文件/README 的相关段落。
