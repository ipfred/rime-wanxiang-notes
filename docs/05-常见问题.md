# 常见问题（部署/不生效/看不懂）

## Q1：我改了 `custom/` 目录里的文件，为什么不生效？

因为 `custom/` 通常只是“携带样例/初始化来源”，真正生效位置是 **你的 Rime 用户目录根目录**。

做法：

- 把“方案包/源仓库”的 `custom/wanxiang.custom.yaml` 复制到用户目录根目录（文件名仍叫 `wanxiang.custom.yaml`）
- 部署一次

## Q2：输入 `/flypy` 提示切换了，但还是没变化？

通常是没部署：切换指令会提示“请重新部署”，照做即可。

如果部署了还不行：

- 确认 `lua/` 目录已在用户目录里
- 确认你启用的是对应 schema（标准版 `wanxiang` 或 PRO）

## Q3：部署很慢 / 卡顿 / 安卓报权限异常？

Android 某些前端的用户目录位于受限路径（如 `data`），直接用第三方文件管理器复制可能导致权限不一致。

优先用系统文件框架/前端提供的导入方式迁移文件，再部署。

## Q4：我想“更新万象”，但又怕覆盖自己的配置怎么办？

原则：

- 你的改动尽量集中在 `*.custom.yaml`、自定义词库、同步目录
- 更新时尽量只替换“官方文件”，不要覆盖你自己的 custom / userdb / sync

源 README 里提到过“all-in-one 工具/白名单保护”的思路；后续本项目会把“安全更新”整理成单独章节（见 Roadmap）。
